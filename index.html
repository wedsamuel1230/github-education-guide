🏆 學生免費申請指南

完全免費

無需綁定信用卡，無需付費，驗證通過即刻享受完整功能。

無限使用

聊天互動、高級請求 (GPT-4o / Claude) 均無次數限制。

有效期長

申請六步走 (以 VTC 為例)

重要提示： 申請全程禁止使用代理 (VPN) 並必須保持瀏覽器或設備的定位權限開啟。 建議：盡量使用手機操作，並在學校網路環境或地理位置附近進行。
1

資格確認與文件準備

確認您年滿 13 歲並在 VTC 等教育機構就讀。在 GitHub 設定中，強制啟用雙因素認證 (2FA)。

準備文件 (四選一)：
  • 學校發出的 `edu.hk` 郵箱地址（優先）。
  • 有效的 VTC 學生證（正面及反面）。
  • 帶有 VTC 名稱、您姓名及日期的在學證明文件。
  • VTC 錄取通知書（包含學校名稱、入學日期）。
2

設定 GitHub 雙重驗證 (Two-Factor Authentication)

⚠️ 重要：申請前必須完成

前往 Settings > Password and authentication > Two-factor authentication 啟用 2FA。

  • 可使用驗證器應用程式 (如 Google Authenticator、Microsoft Authenticator) 或 SMS。
3

完善 GitHub 帳單資訊

⚠️ 重要：申請前必須完成

前往 Settings > Billing and plans > Payment information：

  • 填寫您的全名 (Full Name)，必須與證明文件上的姓名完全一致。
  • 如果您只有單一法定姓名，請在名字 (First Name) 和姓氏 (Last Name) 欄位中均填入相同的名字。
  • 無需新增付款方式 (Payment method)。
4

填寫申請表第一頁

訪問 github.com/education 點擊 "Join GitHub Education"：

  • 郵箱： 必須選擇您的 VTC 學生郵箱 (edu.hk)。
  • 學校名稱： 填寫完整英文名稱，例如：Vocational Training Council,Hong Kong (不要填縮寫 VTC)。
5

上傳證明文件 (建議手機操作)

GitHub 建議使用實時攝像頭拍攝您的文件 (例如學生證/在學證明)，以避免被系統標記為文件被修改而拒絕。

  • 在網頁上選擇 "Take a picture"。
  • 確保文件上的 學校名稱、您的姓名和日期 清晰可見。
6

填寫 VTC 具體信息

填寫學校網站、辦學性質及地址（建議英文）：

  • 學校網站： 填寫官方網址 (如：`www.vtc.edu.hk`)。
  • 辦學性質： 選擇 Higher-education (高等教育)。
  • 地址： 填寫您的校區地址，Country 選擇 China，Non-US State 選擇 Hong Kong。
  • *如果出現地圖，請在左下方搜索框輸入學校全名並選擇地址。*

提交申請與靜候佳音

點擊綠色 "Process my application" 按鈕提交。您將會收到一封確認郵件，審核結果通常在 3-7 個工作日內發送到您的郵箱。

GitHub Student Pack 常見問題

常見拒絕原因：

  • 學術證明文件不清晰：上傳的圖片日期或時間表不符合資格要求，或圖片模糊無法辨識。建議重新上傳清晰的學生證、成績單或其他含有明確日期的證明文件。
  • 使用未驗證域名的學校郵箱：如果學校域名未經 GitHub 驗證，需額外提供學術證明（學生證、成績單等）。
  • 學校郵箱政策寬鬆：某些學校在學生正式註冊繳費前就發放郵箱，這種情況需要額外證明文件。

快速配置示例

以下為在 VS Code / Copilot 中啟動多個 MCP 伺服器的統一配置範例 (概念性示例，依實際套件名稱調整)：

{
    "inputs": [
        { "password": true, "id": "brave-api-key", "type": "promptString", "description": "Brave Search API Key" }
    ],
    "mcp": {
        "servers": {
            "context7": {
                "type": "http",
                "url": "https://mcp.context7.com/mcp",
                "headers": { "CONTEXT7_API_KEY": "YOUR_API_KEY" }
            }
            "sequential-thinking": {
            "command": "npx",
            "args": ["-y", "@modelcontextprotocol/server-sequential-thinking"]
            },
            "filesystem": {
                "command": "npx",
                "args": ["-y", "@modelcontextprotocol/server-filesystem", "${workspaceFolder}"]
            },
            "brave-search-mcp-server": {
                "command": "npx",
                "args": ["-y", "@brave/brave-search-mcp-server", "--transport", "stdio"],
                "env": { "BRAVE_API_KEY": "${input:brave-api-key}" }
            }
        }
    }
}
  • 先執行 npm install 或使用 npx 直接呼叫未全局安裝的套件。
  • Brave Search 需要申請免費 API Key 並放入環境變量 BRAVE_API_KEY。
  • Filesystem MCP 建議限制 ROOT_PATH 為專案根目錄以避免過度訪問。
  • Sequential Thinking 有助於長鏈式推理與架構決策，降低跳躍式回答。
  • Context7 改善大型代碼庫跨檔案引用的解析與語境檢索。

✨ 提示語工程：PIRCAI 框架

好的提示語是讓 AI 產出高品質代碼的關鍵。請遵循 PIRCAI 框架來構建你的請求。

  • P
    Profile (角色)：設定 AI 的身份（如：Python 專家）。
  • I
    Input (輸入)：提供背景信息和現有代碼。
  • R
    Request (請求)：清晰具體的動詞（修復、編寫、優化）。
  • C
    Constraints (約束)：技術棧、性能要求。
  • A
    Assistants (協助)：指定輸出格式（包含註釋、文檔）。
  • I
    Instructions (指令)：逐步說明步驟。
Prompt Example 
# Profile
你是一位 Python 數據處理專家。

# Input
我有一個包含雜亂數據的 CSV 文件，其中包含 'price' 欄位（字符串格式，含貨幣符號）。

# Request
請編寫一個函數 `clean_price_data` 來清洗這些數據。

# Constraints
- 使用 Pandas 庫。
- 必須處理空值（用 0 填充）。
- 性能優先，使用向量化操作。

# AI Assistants
- 為函數添加 Google Style 的 Docstring。
- 在關鍵邏輯處添加行內註釋。

# Instructions
1. 讀取數據。
2. 清洗字符。
3. 轉換類型。
4. 返回處理後的 DataFrame。
